01 — Foundation

Design Principles

Sporza's DNA translated into code. Five non-negotiable rules that govern every pixel.

⚡
Speed above all
Content first, navigation second, decoration never. Users want scores immediately.
🔴
Red is sacred
Only for: LIVE indicators, primary actions, breaking updates, critical alerts. Never decorative.
🃏
Cards everywhere
Consistent padding, borders, corners. Modular blocks that compose predictably.
🔢
Numbers get priority
Scores use font-weight: 800 and tabular-nums. Always.
🫧
Whitespace is a feature
Generous spacing, clean rhythm, clear hierarchy. Not wasted space — intentional breathing room.

11 — Voice

Editorial Tone

Concise. Factual. Immediate. No sensational fluff. Write like a sports wire, not a tabloid.

Headlines
"An incredible and breathtaking match left fans in absolute awe…"
✓ "Club Brugge wint met 2-1 na late treffer in minuut 89."
"In a stunning turn of events, the Belgian Red Devils managed to overcome…"
✓ "België klopt Nederland 3-1 in oefenduel."
UI Copy
"Your request has been successfully processed and saved to the system."
✓ "Event saved."
"Oops! Something went wrong. We're terribly sorry for the inconvenience."
✓ "Save failed — try again."
Key Rules
✔ Short, active, result-focused headlines
✔ Factual — state what happened, not how it felt
✔ Direct — no hedging, no qualifiers
✔ Dutch where possible for Belgian audience, English for system UI
✗ No exclamation marks in UI copy
✗ No "Oops" / "Whoops" / cutesy error messages
✗ No adjective soup — one fact per sentence
12 — Guardrails

Do & Don't

Enforce everywhere. No exceptions. No "but the stakeholder said…"

✓ Do

Use red exclusively for LIVE states, primary actions, and critical alerts.

✗ Don't

Scatter red across the UI for decoration, labels, or secondary information.

✓ Do

Use tabular-nums and heavy weight for scores. Numbers deserve visual priority.

✗ Don't

Show scores in regular weight or proportional figures. A "1" and "8" must occupy same width.

✓ Do

Keep animations fast, subtle, purposeful: live ticks, hover highlights, score transitions.

✗ Don't

Add fancy easing, parallax, or entrance animations. Nobody watches football for CSS transitions.

✓ Do

Generous whitespace, clean vertical rhythm, consistent card padding across all views.

✗ Don't

Pack content into dense grids without breathing room. Density ≠ usefulness.

✓ Do

Use dark mode for video contexts, live dashboards, and broadcast monitoring views.

✗ Don't

Apply decorative gradients unless it's a video hero or a live highlight strip.

✓ Do

Prioritize mobile: live scores → key moments → short summaries → video highlights.

✗ Don't

Ship desktop-first layouts and shoehorn them into mobile. The phone is where urgency lives.
