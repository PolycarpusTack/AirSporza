How the system resolves a multi-competition weekend
① Query Resolution Flow
1
Scheduler opens weekend view for channel Sports 1, date range 2026-03-06 → 2026-03-09
2
API fires: SELECT * FROM fixture WHERE round_id IN (rnd_epl_gw29, rnd_bun_md25, rnd_jpl_md28) ORDER BY scheduled_kickoff_utc — all three leagues, one query
3
JOIN on team (home + away), venue, round → season → competition — fully denormalized for the grid payload
4
Rights engine evaluates each fixture against channel rights policy. Returns LIVE, DELAYED, or BLOCKED per fixture × channel
5
Overlap detector scans for fixtures whose [kickoff_utc, kickoff_utc + broadcast_duration] intervals intersect. Flags Saturday 15:00 UTC as 3-way overlap
6
ResolvedSchedule view returned to UI with validation badges (CONFLICT · RIGHTS · OK) pre-computed per slot. Zero N+1 queries
③ Conflict Resolution Options (Sat 15:00)
Decision required by scheduler
Three events collide at 15:00 UTC on Saturday: Anderlecht–Brugge, Bundesliga 5-pack at 13:30 UTC (still live), and 6× PL 15:00 kick-offs. System presents options:
OptionActionRights impactViewer impact
A Show Anderlecht–Brugge live on Sports 1. Delay PL 15:00 to +90min highlights. JPL live ✓ PL fans miss live
B Show PL 15:00 live on Sports 1. Move Anderlecht–Brugge to Sports 2. Check Sports 2 rights Split audience
C Show PL 15:00 on Sports 1. Delay Anderlecht–Brugge to 19:00 (delayed broadcast). Delay rights required JPL delayed 4hrs
D Run Anderlecht–Brugge live on Sports 1, PL 15:00 live on Sports 2, BUN highlights at 22:00. Cleanest split Ideal but needs 2 live rights
System does not auto-decide. It presents this options panel and requires scheduler confirmation. Decision is logged to audit trail with mandatory reason code.

④ What the planning grid query returns
GraphQL payload (simplified) for the full weekend:
query WeekendPlan {
  fixtures(
    rounds: ["rnd_epl_gw29","rnd_bun_md25","rnd_jpl_md28"]
    channel: "channel_sports1"
  ) {
    id
    kickoff_utc
    kickoff_local  # computed from venue_timezone
    broadcast_duration_min
    competition { id shortCode color }
    round { label roundNumber }
    homeTeam { name abbreviation primaryColor }
    awayTeam { name abbreviation primaryColor }
    venue { name timezone }
    rights { status canBroadcastLive canDelay }
    conflicts {  # pre-computed overlap detection
      conflictsWith { id homeTeam { name } }
      overlapMinutes
    }
    validationBadges { code severity message }
  }
}
Single query. All three leagues. Rights pre-evaluated. Conflicts pre-detected. The scheduler receives a fully decorated payload — no follow-up fetches needed to render the planning grid.
